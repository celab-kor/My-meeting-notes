회의 목적

핵심 목표: 화공플랜트 라인 리스트 자동화 솔루션의 초기 컨셉 설계 및 팀 간 이해 동기화

이 회의는 AI 기반 라인 리스트 자동 생성 솔루션의 UI/UX 설계안을 공유하고, 실제 현장 요구사항과의 적합성을 검증하는 것을 주목적으로 하였습니다. 특히 서로 간의 이해를 동기화하고 실무진의 피드백을 통해 개발 방향성을 명확히 하는데 중점을 두었습니다.

안건 1: UI/UX 설계 및 사용자 워크플로우 ▼

현재 제안된 UI 설계

상윤님이 제시한 초기 UI 설계는 다음과 같은 구조를 가지고 있습니다:

워크플로우:
파일 업로드 → AI 자동 완성 → 셀별 검토 및 설명 → Next Cell 버튼으로 순차 확인

입력 데이터 구성

  • 라인 리스트 Excel 파일: 작업 대상이 되는 기본 템플릿
  • P&ID 및 PFD 파일: 라인 식별을 위한 설계도 데이터
  • 장비/인스트루먼트 데이터시트: 파이프 스펙 결정을 위한 참조 데이터
주요 인사이트:
  • P&ID는 모든 정보의 기준점이며, 변경사항의 주요 원인
  • 라인 번호 체계는 P&ID 번호 + 라인 순번으로 구성
  • 작업 순서에 대한 특별한 관례는 없어 랜덤 순서도 무방
  • 데이터시트의 수치 표기법(유럽식 vs 한국식) 차이점 고려 필요

사용자 피드백 및 개선 방향

결정사항: 과도한 설명 제거, 완성품 우선 제공 방식으로 변경

현장 전문가들의 의견에 따르면, 셀별 상세 설명보다는 완성된 라인 리스트를 먼저 제공하고, 문제가 있는 부분만 별도로 표시하는 방식이 더 효율적입니다.

안건 2: 기술적 요구사항 및 데이터 처리 ▼

라인 리스트 작업의 본질

라인 리스트 작업은 기본적으로 단순 노동의 성격을 가지며, 복잡한 계산보다는 정확한 데이터 매칭이 핵심입니다.

핵심 작업 프로세스:
1. 장비 간 연결 라인 식별
2. 각 라인별 고유 번호 부여
3. 데이터시트에서 해당 스펙(크기, 재질, 압력등급 등) 추출
4. Excel 매크로를 통한 자동 계산 (라인 번호 조합 등)

데이터 정확도 요구사항

현장 전문가 의견:
  • 이 업무는 100% 정확도가 요구되는 작업
  • 실수가 있어도 수정 가능하지만, AI 도구라면 높은 정확도 기대
  • 가져올 수 있는 데이터 소스가 명확히 정해져 있어 기술적 구현 가능
  • OCR 기반 텍스트 인식으로도 충분히 달성 가능한 수준

버전 관리 및 변경 사항 대응

P&ID는 프로젝트 진행 중 빈번하게 변경되며(하루에도 여러 번), 이에 따른 라인 리스트 업데이트가 필요합니다. 각 회사마다 고유한 버전 관리 체계를 운영하고 있습니다.

안건 3: 정확도 및 품질 보증 ▼

법적 책임 및 품질 기준

중요 결정: 상품화 시 99.9% 이상의 정확도 달성 필요

AI 도구가 생성한 결과물을 최종본으로 제출할 경우, 법적 책임이 따르므로 매우 높은 정확도가 요구됩니다.

오류 처리 및 검증 방안

  • 완성본 우선 제공: 전체 라인 리스트를 먼저 완성 후 제공
  • 오류 표시: 문제가 있는 셀만 색상으로 구분하여 표시
  • 간단한 오류 정보: "다르다" 정도의 간단한 피드백만 제공
  • 원본 데이터 참조: 어떤 데이터를 참조했는지 추적 가능하도록 구성
현장 실무진 요구사항:
  • 과도한 설명보다는 시간 단축이 최우선
  • 틀린 부분만 명확히 표시하면 엔지니어가 직접 확인 가능
  • AI의 사고 과정 설명은 불필요 (단순 매칭 작업이므로)

주요 결정사항

1. UI/UX 설계 방향 변경
기존의 셀별 상세 설명 방식에서 완성품 우선 제공 + 오류 표시 방식으로 전환
2. 개발 우선순위 설정
완성도 높은 라인 리스트 자동 생성을 1차 목표로 설정, 부족한 데이터 상황은 2차 고려사항
3. 정확도 목표 설정
상품화 시 99.9% 이상의 정확도 달성을 목표로 하며, 법적 책임을 고려한 품질 보증 필요
4. 회의 일정 조정
빠른 개발 진행을 위해 평일 단시간 회의 + 주말 장시간 회의 병행
